📌 Scripts disponíveis
No diretório do projeto, você pode executar os seguintes comandos:

npm start  
Executa o app em modo de desenvolvimento.
Abra http://localhost:3000 no navegador para visualizar.

A página recarrega automaticamente quando você faz alterações.

Erros de lint aparecerão no console.

npm test  
Inicia o runner de testes no modo interativo.
Veja a seção sobre execução de testes para mais informações.

npm run build  
Cria a versão de produção do app na pasta build.

O React é empacotado em modo de produção.

Os arquivos são minificados e otimizados para melhor desempenho.

Os nomes dos arquivos incluem hashes para cache eficiente.
Seu app estará pronto para ser deployado!

npm run eject  
⚠️ Atenção: esta operação não pode ser desfeita!

Remove a dependência única do projeto.

Copia todos os arquivos de configuração (Webpack, Babel, ESLint, etc.) para dentro do projeto.

Permite controle total sobre as configurações.

Todos os comandos continuam funcionando, mas agora apontam para os scripts copiados.

Use apenas se realmente precisar personalizar profundamente o projeto.

💡 Você nunca é obrigado a usar eject. Para a maioria dos projetos pequenos e médios, o conjunto de recursos padrão já é suficiente.
